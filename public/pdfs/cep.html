Complex Event Processing (CEP)
Made Simple

Write SQL. Detect Patterns. React in Real-Time.

SELECT userId, count(DISTINCT location) as locations, sum(amount) as total FROM hop(transactions, 5s, 10m)
GROUP BY window_start, userId HAVING locations > 1 AND total > 1000

That's it! No complex syntax. Just SQL.

What is Complex Event Processing?

CEP enables real-time analysis and detection of patterns, relationships, and trends across multiple streams of data events.

📊 Common Patterns:

• sequences – Event A followed by B then C within 10 minutes 🔄

• spatial relationships – 100+ transactions/second from same user ⚡

• statistical patterns – Events in same geographic region 📍

😫 The Traditional Way (Flink CEP)

SELECT T.aid, T.bid, T.cid FROM MyTable MATCH_RECOGNIZE ( PARTITION BY userid ORDER BY proctime MEASURES A.id AS aid, B.id AS bid, C.id AS cid PATTERN (A B C) DEFINE A AS name = 'a', B AS name = 'b', C AS name = 'c' ) AS T
requiring a specialized query language, many known limitations

The Timeplus Way

Advanced CEP Patterns

1️⃣ Sequence Patterns

Ordered event chains (A→B→A)

2️⃣ Threshold Patterns

Count-based detection (≥3 A's within 5 seconds)

3️⃣ Absence Patterns

Negative logic (A without C occurring)

4️⃣ Alternating Patterns

Repeating sequences (A-B-A-B...)
